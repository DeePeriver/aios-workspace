# Guide

経営者がClaude Codeとaios-workspaceプラグインを理解し、最初の一歩を踏み出せるように案内する教育的スキル。

## 前提

ユーザーは以下のいずれにも該当する可能性がある：
- Claude Codeが何なのか分からない
- 「スキル」「スラッシュコマンド」「プラグイン」という言葉を聞いたことがない
- なぜタスクを「計画」する必要があるのか、「コンテキスト」を整理する意味が分からない
- 「コマンド」「ターミナル」のような開発者用語に馴染みがない

すべてを **ゼロから** 説明する。専門用語を使う場合は必ずその場で噛み砕く。経営者を子ども扱いはしないが、初めての概念は丁寧に。

## 出力構成

以下の **5部構成** で、必ず上から順に出力する。たとえユーザーが既知でも、`guide` を呼び出した = 確認したい時なので、毎回フル表示する。

---

### 第1部：このプラグインは何か（必ず最初に表示）

以下の3点を、それぞれ2〜4文の平易な日本語で説明する。

1. **Claude Codeとは何か**
   - Anthropic社が作ったAI「Claude」と対話しながら作業を進めるツール。
   - 普通のChatGPT的なチャットと違って、自分のコンピュータのファイルを実際に見たり、書いたり、調べたりできる。
   - つまり「相談相手」と「実際に手を動かしてくれるアシスタント」を兼ねている。

2. **aios-workspaceプラグインとは何か**
   - Claude Codeを「経営者の事業パートナー」として使えるようにする拡張機能。
   - あなたの事業のこと（顧客、戦略、財務、優先事項など）を一度Claudeに教えれば、毎回説明し直さなくても理解した状態で会話ができる。
   - 単発の質問だけでなく、計画作り・実行・振り返りといった日常の業務サイクル全体をサポートする。

3. **どう使うか（スラッシュコマンドの説明）**
   - 入力欄に `/`（スラッシュ）から始まる文字を打つと、「スキル」と呼ばれる特定の作業をClaudeにお願いできる。
   - 例: `/aios-workspace:setup` と打つと、初回セットアップのスキルが動く。
   - スキルを使わず、普通の質問や相談を日本語で話しかけても問題なく動く。スキルは「決まった作業を確実にやってもらう」ための便利なショートカット。

---

### 第2部：使えるスキル一覧と、なぜ使うか

7つのスキルを、以下のフォーマットで **必ず全て** 紹介する。順序も以下の通り：

```
**`/aios-workspace:<名前>`**
- 何をする: [1〜2文]
- なぜ使う: [価値・解決する問題]
- こんな時に使う: [具体的なシーン例]
```

#### 1. `/aios-workspace:setup` — 最初のセットアップ
- **何をする**: あなたの事業のこと（業種、顧客、戦略、財務、現在の優先事項など）をインタビュー形式で聞き出し、Claudeに覚えさせる。
- **なぜ使う**: 経営の話をClaudeにする時、毎回「うちは○○な会社で、ターゲット顧客は××で、今は△△に注力していて…」と前置きするのは大変。一度教えれば次回以降は自動で読み込まれ、すぐに本題に入れる。
- **こんな時に使う**: 初めてこのプラグインを使う時。1回だけ実行する（30分程度かかる）。

#### 2. `/aios-workspace:plan` — 計画作り
- **何をする**: 何かをやる前に、ゴール・ステップ・成功条件・リスクを整理した計画書を作る。
- **なぜ計画が必要か**: 「すぐ着手できる小さなタスク」と「ちゃんと考えてから動くべき大きなタスク」がある。後者をいきなり始めると、抜け漏れや手戻りが発生しやすい。先に計画を文書化しておけば、自分の頭も整理されるし、進捗確認や振り返りもしやすく、人に任せる時もスムーズ。
- **こんな時に使う**: 「来月新サービスを出したい」「営業担当を採用したい」「価格を見直したい」「マーケ予算の使い道を決めたい」など、すぐ着手するには大きすぎるタスクの前に。

#### 3. `/aios-workspace:execute` — 計画の実行
- **何をする**: `plan` で作った計画書を開き、ステップごとにClaudeと一緒に進めていく。各ステップで必要な調査やドラフト作成をClaudeが手伝う。
- **なぜ使う**: 計画は作っただけでは価値が出ない。実行する時に Claude が手を動かしてくれるので、進めるスピードが上がる。
- **こんな時に使う**: 計画を作った後、実際に動き出す時。

#### 4. `/aios-workspace:brainstorm` — 一緒に考える
- **何をする**: 「どうしようかな」と迷っているテーマについて、複数の選択肢と各々のトレードオフを出してくれる。すぐに結論を出さず、視点を広げるための時間。
- **なぜ使う**: 経営者は孤独で、話し相手がいないことも多い。Claudeはあなたの事業を理解した上で、客観的な「壁打ち相手」になれる。1人で考えるより視点が増える。
- **こんな時に使う**: 「価格をどう設定すべきか」「この新規事業、やるかやらないか」「次に採用するならどんな人」など、答えが1つに決まらない問いがある時。

#### 5. `/aios-workspace:update-context` — 情報の更新
- **何をする**: setupで一度教えた事業情報を、変化に合わせて更新する。新しい顧客の追加、戦略の見直し、財務状況の変化など。
- **なぜ使う**: 事業は動いている。半年前の情報のままだとClaudeのアドバイスもズレてくる。情報を新鮮に保つことで、的確な相談相手であり続けられる。
- **こんな時に使う**: 月1回の振り返り、四半期の節目、または何か大きな変化があった直後。

#### 6. `/aios-workspace:audit-workspace` — 健康診断
- **何をする**: 事業情報の整理状況をチェックし、古くなった項目や抜けている情報を指摘して、改善案を出す。
- **なぜ使う**: 自分では気づきにくい情報の抜けや古さを発見できる。定期的にやることでClaudeとの対話の質が保てる。
- **こんな時に使う**: 久しぶりにClaude Codeを使う時。または数ヶ月に1回の定期チェック。

#### 7. `/aios-workspace:guide` — このガイド
- **何をする**: 今あなたが見ている、このガイドをもう一度表示する。
- **こんな時に使う**: 何ができるか忘れた時、操作に迷った時。いつでも戻ってきていい。

### 第3部：あなたは今どこにいるか

作業ディレクトリを確認し、4つの状態のどれに当てはまるか判定する：

| 状態 | 検出方法 | 意味 |
|---|---|---|
| **空** | `CLAUDE.md` も `context/` も無い | 何もセットアップされていない真っさらな状態 |
| **作業中** | 一部のファイルはあるが、必須の7ファイルが揃っていない、または `{{...}}` プレースホルダーが残っている | setupが途中で中断された状態 |
| **準備完了** | `CLAUDE.md` があり、`context/` 内の7ファイルがすべて記入済み | いつでも仕事を始められる状態 |
| **古い** | 準備完了だが、ファイルの `Last updated:` が古い（`current-priorities` / `financials` は30日以上前、その他は90日以上前） | 情報の更新が必要な状態 |

判定結果と1〜2行の補足を伝える：
- **空** → 「まだ何もセットアップされていません。事業情報がClaudeに登録されていない状態です。」
- **作業中** → 「セットアップが途中で止まっています。」 + 何が記入待ちか
- **準備完了** → 1行サマリー（事業名 + 現在の最優先事項）
- **古い** → 1行サマリー + どのファイルがいつから更新されていないか

判定に必要な分だけファイルを読む。全ファイルの全文は読まない。

---

### 第4部：最初の一歩（状態別）

状態に応じて、最も自然な次の行動を **1つだけ** 提案する：

| 状態 | 提案する次の行動 |
|---|---|
| **空** | `/aios-workspace:setup` を実行してください。事業情報をClaudeに教え込む最初のステップです（30分程度）。 |
| **作業中** | `/aios-workspace:setup` を再実行すれば、中断したところから続けられます。 |
| **準備完了** | 何でも話しかけてください。やりたいことが明確なら `/aios-workspace:plan`、迷っていることがあるなら `/aios-workspace:brainstorm` が自然な入口です。 |
| **古い** | まず `/aios-workspace:audit-workspace` で状態を確認するか、古いファイルを `/aios-workspace:update-context` で更新してください。 |

---

### 第5部：尋ねる

最後にこう締めくくる：

> 何をしたいですか？ 上記のスキルから選んでもいいですし、普通の日本語で「○○について相談したい」「△△を計画したい」のように話しかけてもらっても構いません。

それ以上のことを進めない。このスキルはあくまで案内役。

---

## トーン

- 専門用語は最小限。使う場合は必ずその場で噛み砕く（例:「スラッシュコマンド = `/` から始まる文字列のこと」）。
- 1段落は2〜4文まで。長文の壁を作らない。
- 「できます」より「○○な時に使います」のほうが具体的で記憶に残る。
- 経営者をプロとして扱うが、ITに馴染みのない人にも伝わる平易さを保つ。

## やってはいけないこと

- 第1部・第2部を省略しない。たとえユーザーが既知でも、`guide` を呼び出した = 確認したい時なので毎回フル表示する。
- 各スキルの説明から「なぜ使うか」を削らない。これがこのスキルの中核。
- 状態判定（第3部）だけを返してメニューを省略しない。
- 推測で次のスキルを自動実行しない。最後に必ずユーザーに尋ねる。
