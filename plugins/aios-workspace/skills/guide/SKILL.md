# Guide

ユーザーを方向づける。何ができるか、どこにいるか、次に何をするかを見せる。

## 手順

### 1. 現状を確認

作業ディレクトリを確認し、どのバケツに当てはまるかを報告する：

| 状態 | 検出 | 意味 |
|---|---|---|
| **Empty（空）** | `CLAUDE.md` も `context/` も無い | 完全に新規 — 何もセットアップされていない |
| **Partial（部分的）** | 一部のファイルはあるが必須次元が欠けている、またはファイルに `{{...}}` プレースホルダーが残っている | セットアップが始まったが完了していない |
| **Ready（準備完了）** | `CLAUDE.md` が存在し、`context/` 内の7つのコンテキストファイルがすべて記入済み | 完全にセットアップ済み |
| **Stale（古い）** | Readyだが、1つ以上のコンテキストファイルの `Last updated:` が90日（または `current-priorities` / `financials` の場合は30日）より古い | セットアップ済みだがリフレッシュが必要 |

判定に必要な分だけ読む — 全ファイルの全文を読まない。

### 2. 利用可能なスキルを一覧表示

何ができるかのコンパクトなメニューをユーザーに見せる。次の正確な形式と順序を使う：

```markdown
## できること

**セットアップとメンテナンス**
- `/aios-workspace:setup` — 新規ワークスペースの初回ブートストラップ
- `/aios-workspace:update-context <内容>` — ビジネスコンテキストを追加・更新（新しい事実の追加、領域のリフレッシュなど）
- `/aios-workspace:audit-workspace` — ワークスペースの健全性をチェック

**日常使用**
- `/aios-workspace:plan <task>` — やる前に計画する
- `/aios-workspace:execute <plan-path>` — 保存された計画を実行する
- `/aios-workspace:brainstorm <topic>` — オープンな質問を探索する
- `/aios-workspace:guide` — このメニューをもう一度表示する
```

### 3. 現在地を見せる

短いステータスブロックを表示する：

```markdown
## 現在地

**ディレクトリ:** <作業ディレクトリの絶対パス>
**ワークスペース状態:** <Empty / Partial / Ready / Stale>
<状態に関連した1〜2行の詳細 — 下記参照>
```

状態別の詳細：

- **Empty:** 「ワークスペースファイルが検出されませんでした。`/aios-workspace:setup` から始めてください。」
- **Partial:** 7つのコンテキストファイルのうちどれが欠けているか、またはプレースホルダーが残っているかを一覧表示する。`/aios-workspace:setup` を再開のために提案する。
- **Ready:** 1行でサマリー — ビジネス名（`business-overview.md` から）、最優先事項（`current-priorities.md` から）。
- **Stale:** Readyと同じ1行サマリーに加え、どのファイルが古くいつ更新されたかをリスト化する。

### 4. 次のアクションを推奨

状態に基づいて、最も役立つ次のスキルを1つ提案する：

| 状態 | 推奨される次 |
|---|---|
| Empty | `/aios-workspace:setup` |
| Partial | `/aios-workspace:setup`（再開）または `/aios-workspace:audit-workspace` |
| Ready | 推奨を強制せず、何をしたいか尋ねる（`/aios-workspace:plan` または `/aios-workspace:brainstorm` が一般的） |
| Stale | 一番古いファイルに対して `/aios-workspace:update-context <area>`、または `/aios-workspace:audit-workspace` |

### 5. 尋ねる

直接的でオープンな質問で締めくくる：「何をしたいですか？」

それ以外のことを進めない。このスキルはナビゲーション専用。

## やってはいけないこと

- 他のスキルを自動実行しない。ユーザーはもっとシンプルな何かをやりたいかもしれない。
- ファイルの全文を貼り付けない。ステータスブロックは10行以下に抑える。
- 状態を推測しない — チェックが曖昧なら、より低い確信度の状態（ReadyよりもPartial）を報告する。
