# Audit Workspace

ワークスペースをチェックリストに照らして評価し、ギャップと具体的な次のアクションを報告する。

## 実行するチェック

### 構造
- ワークスペースルートに `CLAUDE.md` が存在する
- `context/`、`plans/`、`outputs/`、`reference/` のディレクトリが存在する

### コンテキストファイル
7つの次元それぞれについて、ファイルが存在し実コンテンツ（テンプレートだけではない）を含むことを確認する：
- `context/business-overview.md`
- `context/personal-profile.md`
- `context/strategy.md`
- `context/customers.md`
- `context/competitors.md`
- `context/financials.md`
- `context/current-priorities.md`

### 鮮度
- 各コンテキストファイルの `Last updated:` 行を読む。
- 90日より古い場合は **古い** としてフラグを立てる。
- `strategy.md` は60日より古い場合に古いとフラグ。
- `current-priorities.md` は30日より古い場合に古いとフラグ。
- `financials.md` は30日より古い場合に古いとフラグ。

### 完全性
各コンテキストファイルについて以下をスキャンする：
- 記入されていない `{{...}}` プレースホルダー
- `TODO:` マーカー
- 見出し直下に内容がない空のセクション

### 整合性
ファイル間で矛盾がないかチェックする：
- `strategy.md` の目標が `current-priorities.md` のどこにも反映されていない
- `strategy.md` で言及された顧客セグメントが `customers.md` に存在しない
- `strategy.md` または `customers.md` で名指しされた競合が `competitors.md` に存在しない
- `strategy.md` または `current-priorities.md` で参照された財務指標が `financials.md` に存在しない

## 出力形式

3つのセクションのマークダウンレポートを生成する：

```markdown
## ワークスペース監査 — <日付>

### 機能している点
- <グリーンチェック、1行ずつ>

### ギャップ
- **<ファイルパス>** — <何が問題か> → <1行の修正案>
- ...

### 推奨される次のアクション
1. <最優先の修正> — `/aios-workspace:<skill> <args>` を実行
2. <次>
3. ...
```

**推奨される次のアクション** はインパクト順に並べる：
1. ファイル欠落が最優先（他のスキルがコンテキストを読めない）
2. 古い戦略 / 優先事項が次（現在の作業を駆動する）
3. 古い財務情報が3番目
4. 整合性ギャップが4番目
5. スタイルや完全性のギャップが最後

## 終了

レポートを生成した後、最優先のギャップを今すぐ修正したいかユーザーに尋ねる。

## やってはいけないこと

- 自動修正をしない。このスキルは報告し、決めるのはユーザー。
- 必須でないオプションのセクション欠落をギャップとしてフラグしない。必須次元の欠落のみフラグする。
