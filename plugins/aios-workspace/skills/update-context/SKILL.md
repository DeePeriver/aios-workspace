# Update Context

ワークスペースの `context/` 内のファイルを編集する。

## 手順

### 1. 対象ファイルを特定

ユーザーの要求が以下の7次元のどれに該当するか判定する：

| 事実の対象 | 行き先 |
|---|---|
| ビジネスの内容、構造、チーム | `context/business-overview.md` |
| オーナー個人 | `context/personal-profile.md` |
| ビジョン、目標、戦略的賭け、選ばないこと | `context/strategy.md` |
| 顧客、セグメント、フィードバック | `context/customers.md` |
| 競合または代替手段 | `context/competitors.md` |
| お金、収益、コスト、ランウェイ | `context/financials.md` |
| 現在取り組んでいること | `context/current-priorities.md` |

該当が不明確な場合は1問だけ確認する。どの次元にも当てはまらない場合は、`context/` 内に新規ファイルを作成することをユーザーに提案し、確認を得てから作成する。

### 2. 必ずファイルを読んでから編集する

書き換える前に既存内容を読む。これは追加・更新・削除すべてのケースで必須。すでにサブフォルダ構造（`context/<dimension>/`）に分割されている場合は、その構造も含めて把握する。

### 3. 編集する

ユーザーの要求に応じて適切に編集する：

- 新しい事実の追加 → 該当セクションに追記
- 特定セクションの書き換え → 既存内容を更新
- ファイル全体のリフレッシュ → テンプレート構造（見出し）を保ったまま書き直し。まだ正確な内容は保持する
- 削除 → 該当箇所を取り除く
- すでにサブフォルダ構造が存在する場合 → その構造を維持しながら該当箇所だけ更新する

既存内容と矛盾する事実が来た場合は、ユーザーに「追加でなく置き換えるか?」を確認する。重複する事実はスキップしてユーザーに伝える。

### 4. ファイルサイズによる自動分割

書き直したファイルが大きくなりすぎた場合（200〜250行以上）、サブフォルダ構造に自動で分割する。

- 対象次元の分割基準とターゲット構造は `${CLAUDE_PLUGIN_ROOT}/skills/update-context/references/<dimension>-split.md` を読み込んで参照する（`<dimension>` はステップ1で特定した次元名）
- 元のファイルは **要約集** として残す — 単なる目次ではなく、各サブセクションに **2〜3文の短いサマリー** を含め、その末尾にサブファイルへのリンクを添える。読者がリンクを開かなくても全体像が掴めるレベル
- 詳細はターゲット構造に従って対応するサブファイルへ移す
- 分割が発生したらステップ6の報告で必ず言及する

### 5. Last updated を更新

ファイル先頭の `Last updated:` を今日の日付に設定する。

### 6. 報告

何がどこにどう変わったか（ファイルパス + セクション、必要なら分割の有無）をユーザーに伝える。行単位の差分ではなく、実質的な変化のみ。

## やってはいけないこと

- 読まずに書き換えない
- 同じ事実を二度追加しない
- 確認なしに `context/` 内に新規ファイルを作らない
- コンテンツを黙って削除しない（削除する場合は報告で言及する）
