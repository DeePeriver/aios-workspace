# strategy 分割戦略

## いつ分割するか
戦略は一貫性のある単一文書として保つのが基本。以下に当てはまる場合のみ分割する：
- ファイル全体が **300行を超える**
- 戦略のスナップショットを過去のものとして残したい（四半期更新時など）
- 個別の戦略的賭けが各 **10行以上** の深掘り記述を持つ
- 戦略決定の根拠やオプション比較を別途保管したい

## ターゲット構造
```
context/strategy.md            ← 現在の戦略（インデックス）
context/strategy/
├── archive-YYYY-Q<n>.md       ← 過去の戦略スナップショット
├── bet-<name>.md              ← 個別の戦略的賭けの深掘り
└── deep-dives/<topic>.md      ← トピック別の戦略的考察
```

## インデックスファイルに残すもの

インデックスは「目次」ではなく「**要約集**」として機能させる。各サブセクションは **2〜3文の短いサマリー** を持ち、必要ならサブファイルへのリンクを添える。

- **3年ビジョン**
- **12ヶ月の目標**
- **各戦略的賭け** — 2〜3文のサマリー（何に賭けているか、なぜか、撤退条件）+ 深掘りファイルへのリンク
- **選ばないこと**
- **なぜこの戦略か** — 2〜3文の根拠
- **過去アーカイブ** — 2〜3文で過去の戦略変遷の概要（最近何が変わったか）+ 最新アーカイブへのリンク

## サブファイルに移すもの
- 過去の戦略のフルテキスト（更新時にアーカイブ化）
- 個別の戦略的賭けについての詳細な根拠、想定、進捗、撤退条件
- トピック別の戦略的考察（市場分析、参入戦略、撤退議論など）

## 命名規則
- アーカイブ: `archive-YYYY-Q<n>.md`（例: `archive-2026-Q1.md`）
- 賭け: `bet-<kebab-case-name>.md`（例: `bet-enterprise-pivot.md`）
- 深掘り: `deep-dives/<kebab-case-topic>.md`（例: `deep-dives/japan-market-entry.md`）

## 分割実行時の注意
- 戦略をアーカイブする時は、現在のファイル全体をスナップショットとして `archive-YYYY-Q<n>.md` にコピーしてから新しい内容で書き直す
- 賭けの深掘りファイルには、いつ・なぜその賭けを始めたか、撤退条件、進捗を必ず含める
- インデックスの先頭に `Last updated:` 行を残す
