# current-priorities 分割戦略

## いつ分割するか
以下のいずれかに当てはまる場合：
- ファイル全体が **200行を超える**
- 「最近完了したもの」が **15項目以上** ある
- 「パーキングロット」が **10項目以上** ある
- 過去の優先事項を四半期ごとに振り返り資料として残したい

## ターゲット構造
```
context/current-priorities.md  ← 現在のアクティブな優先事項（インデックス）
context/current-priorities/
├── completed-YYYY-Q<n>.md     ← 四半期ごとの完了アーカイブ
├── parking-lot.md             ← パーキングロット詳細
└── archive-YYYY-Q<n>.md       ← 過去の優先事項スナップショット
```

## インデックスファイルに残すもの

インデックスは「目次」ではなく「**要約集**」として機能させる。各サブセクションは **2〜3文の短いサマリー** または現在のアクティブな項目リストを持ち、必要ならサブファイルへのリンクを添える。

- **今月の優先事項**（リスト、本文のまま）
- **今四半期の優先事項**（リスト、本文のまま）
- **アクティブなブロッカー**（リスト、本文のまま）
- **直近の完了項目** — 過去30日以内、最大5項目
- **パーキングロット** — 2〜3文で全体傾向（どんなアイデアが滞留しているか）+ 完全リストへのリンク
- **完了アーカイブ** — 2〜3文で過去四半期の総括（何を成し遂げたか）+ 最新アーカイブへのリンク

## サブファイルに移すもの
- 30日以上前に完了した項目（四半期別アーカイブへ）
- 完全なパーキングロット
- 過去の優先事項のスナップショット（四半期締めごと）

## 命名規則
- 完了アーカイブ: `completed-YYYY-Q<n>.md`（例: `completed-2026-Q1.md`）
- パーキングロット: `parking-lot.md`
- 過去アーカイブ: `archive-YYYY-Q<n>.md`

## 分割実行時の注意
- 完了項目をアーカイブに移す時は、完了日を必ず保持する
- 四半期締めの自動化を考慮 — 四半期を跨いだ更新時は完了項目を該当四半期ファイルに移す
- インデックスの先頭に `Last updated:` 行を残す
