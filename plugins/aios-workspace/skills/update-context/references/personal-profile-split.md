# personal-profile 分割戦略

## いつ分割するか
personal-profile は通常、分割不要。以下のいずれかに当てはまる場合のみ分割する：
- ファイル全体が **200行を超える**
- 強み/弱みのリストが各 **10項目以上** に膨らんだ
- ワークスタイルの記述が **15行以上** に長文化
- セルフコーチング用のメモやリフレクション履歴を残したい

## ターゲット構造
```
context/personal-profile.md    ← インデックス
context/personal-profile/
├── working-style.md           ← ワークスタイルの長文版
├── strengths-weaknesses.md    ← 強み/弱みの詳細リスト
└── reflections.md             ← セルフリフレクションのアーカイブ（任意）
```

## インデックスファイルに残すもの

インデックスは「目次」ではなく「**要約集**」として機能させる。各サブセクションは **2〜3文の短いサマリー** を持ち、必要ならサブファイルへのリンクを添える。

- **オーナーの基本情報** — 名前、役割、バックグラウンド
- **ワークスタイル** — 2〜3文のサマリー（核心的な働き方の特徴）+ 詳細へのリンク
- **強み/弱み** — 各上位3項目を短く + 詳細リストへのリンク
- **使える時間**
- **エネルギーが湧くこと/消耗すること** — 短ければ本文のまま、詳細があればリフレクションへのリンク

## サブファイルに移すもの
- 詳細なワークスタイル記述（働く時間帯、集中の仕方、苦手な状況など）
- 完全な強み/弱みリスト
- セルフリフレクション、コーチングメモ、過去の自己評価

## 命名規則
- ワークスタイル: `working-style.md`
- 強み/弱み: `strengths-weaknesses.md`
- リフレクション: `reflections.md`

## 分割実行時の注意
- このファイルは個人的内容を含むため、リポジトリを共有する場合は注意
- インデックスの先頭に `Last updated:` 行を残す
