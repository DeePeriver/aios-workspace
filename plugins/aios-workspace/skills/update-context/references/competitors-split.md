# competitors 分割戦略

## いつ分割するか
以下のいずれかに当てはまる場合：
- ファイル全体が **300行を超える**
- 「直接競合」が **4社以上**、かつ各社の記述が10行以上
- 「ウォッチリスト」が **10項目以上** ある
- 競合分析の深掘りや、競合の動向履歴を残したい

## ターゲット構造
```
context/competitors.md         ← インデックス
context/competitors/
├── <competitor-name>.md       ← 競合ごとの深掘り
├── watchlist.md               ← ウォッチリスト
└── differentiation.md         ← 自社差別化の詳細考察
```

## インデックスファイルに残すもの

インデックスは「目次」ではなく「**要約集**」として機能させる。各競合・各サブセクションは **2〜3文の短いサマリー** を持ち、必要なら詳細サブファイルへのリンクを添える。

- **各直接競合** — 2〜3文のサマリー（何をしているか、ポジショニング、自社にとっての脅威度）+ 詳細ファイルへのリンク
- **間接的代替手段** — 2〜3文の概要（顧客が代わりに何をしているか）
- **自社の差別化** — 1〜2文の核心 + 詳細考察があれば `differentiation.md` へのリンク
- **ウォッチリスト** — 2〜3文で全体傾向（どんな企業/トレンドを追っているか）+ 完全リストへのリンク

## サブファイルに移すもの
- 各競合の詳細プロフィール（プロダクト、価格、強み・弱み、最近の動向、自社との比較）
- 完全なウォッチリスト（変動の激しい項目）
- 差別化の詳細な根拠と分析（自社が選ばれる/選ばれない理由のディープダイブ）

## 命名規則
- 競合: `<kebab-case-name>.md`（例: `acme-corp.md`、`competitor-x.md`）
- ウォッチリスト: `watchlist.md`
- 差別化: `differentiation.md`

## 分割実行時の注意
- 競合名のファイル名は固有名詞のkebab-case（日本語の場合はローマ字 or 略称）
- 同名衝突回避のため、必要なら接尾辞を付ける（例: `acme-jp.md`）
- インデックスの先頭に `Last updated:` 行を残す
