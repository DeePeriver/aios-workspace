# financials 分割戦略

## いつ分割するか
以下のいずれかに当てはまる場合：
- ファイル全体が **250行を超える**
- 月次・四半期の財務スナップショットを履歴として残したい
- ユニットエコノミクス（CAC、LTV、ペイバック期間など）に **詳細な計算や推移** が必要
- 主要指標が **10個以上**、かつ各指標に解説が付く
- 投資家向けレポートや会計士向けの整理データを保管したい

## ターゲット構造
```
context/financials.md          ← 現在のスナップショット（インデックス）
context/financials/
├── history-YYYY.md            ← 年次アーカイブ
├── snapshots/YYYY-MM.md       ← 月次スナップショット（任意、頻繁に更新する場合）
├── unit-economics.md          ← ユニットエコノミクスの詳細
└── metrics-glossary.md        ← 指標の定義と解釈
```

## インデックスファイルに残すもの

インデックスは「目次」ではなく「**要約集**」として機能させる。各サブセクションは **2〜3文の短いサマリー**（または最新の数値スナップショット）を持ち、必要ならサブファイルへのリンクを添える。

- **収益モデル** — 2〜3文（どうお金を得ているか、価格構造の核心）
- **現状** — 最新月の数値スナップショット（収益、マージン、バーン/ランウェイ、キャッシュポジション）
- **コスト構造** — 高レベルなブレイクダウン
- **主要指標** — 現在値リスト + 各指標の **1〜2文の意味づけ**（用語集へのリンク）
- **制約**
- **履歴** — 2〜3文で直近の傾向（前年比、四半期傾向）+ 履歴ファイルへのリンク
- **ユニットエコノミクス** — 2〜3文の核心（CAC・LTVの現在値と傾向）+ サブファイルへのリンク

## サブファイルに移すもの
- 過去の月次・年次スナップショット（時系列の数値）
- ユニットエコノミクスの詳細計算と推移（CAC、LTV、ペイバック、ARR推移など）
- 各指標の定義、計算方法、解釈ガイド、ベンチマーク

## 命名規則
- 年次アーカイブ: `history-YYYY.md`（例: `history-2025.md`）
- 月次スナップショット: `snapshots/YYYY-MM.md`（例: `snapshots/2026-04.md`）
- ユニットエコノミクス: `unit-economics.md`
- 指標用語集: `metrics-glossary.md`

## 分割実行時の注意
- 数値の信頼性が重要。サブファイル化する時は集計期間と通貨を明示する
- インデックスの「現状」は **最新月** の数値のみ。過去は履歴ファイルへ
- インデックスの先頭に `Last updated:` 行を残す
