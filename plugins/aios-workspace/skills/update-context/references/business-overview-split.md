# business-overview 分割戦略

## いつ分割するか
business-overview は通常、分割不要。以下のいずれかに当てはまる場合のみ分割する：
- ファイル全体が **200行を超える**
- 「起源・原点」セクションが **10行以上** に膨らんだ
- 沿革やマイルストーンを時系列で残したくなった
- 重要事実が **10項目以上** ある

## ターゲット構造
```
context/business-overview.md   ← インデックス
context/business-overview/
├── history.md                 ← 沿革・起源の長文版
└── milestones.md              ← 重要な出来事のリスト（任意）
```

## インデックスファイルに残すもの

インデックスは「目次」ではなく「**要約集**」として機能させる。各サブセクションは **2〜3文の短いサマリー** を持ち、必要ならサブファイルへのリンクを添える。

- **「何をしているビジネスか」** — 1文
- **ステージ**
- **構造** — 法人形態、チーム規模、拠点、設立年
- **起源・原点** — 2〜3文のサマリー（核心的な創業ストーリー）+ 詳細へのリンク
- **重要事実** — 上位3〜5項目を短く + 詳細リストへのリンク（必要なら）
- **沿革・マイルストーン** — 2〜3文で全体感（どう成長してきたか）+ サブファイルへのリンク

## サブファイルに移すもの
- 起源・原点の長文記述（創業ストーリー、初期の意思決定）
- 沿革（時系列の出来事）
- マイルストーン（資金調達、プロダクトリリース、採用、移転など）
- 詳細な背景情報

## 命名規則
- 沿革: `history.md`
- マイルストーン: `milestones.md`

## 分割実行時の注意
- このファイルは変更頻度が低い。分割しても四半期に1回程度の更新で十分
- インデックスの先頭に `Last updated:` 行を残す
